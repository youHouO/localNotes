1.需求文档MRD/PRD
一、核心诉求
打造一款以数据安全和一致性为绝对底线的个人笔记软件，采用「本地唯一真相源+多网盘镜像备份」的分布式架构，彻底解决传统云端笔记的数据所有权、厂商锁定、同步不可靠问题。软件追求极简、可靠、可控，只为单用户个人写作场景服务，不做任何多余功能，不为极少数用户的特殊需求妥协大多数人的体验。
二、不可动摇的底线原则（所有设计不得违反）
1. 本地绝对优先：所有读写操作100%先走本地，云端仅作为异地备份镜像；无网络时所有功能完全可用，网络恢复后自动补同步；云端故障绝不影响本地使用。
2. 数据非明文：所有本地和云端的笔记与图片均为加密格式，无法直接用其他编辑器打开查看。
3. 零服务器成本与责任：不搭建任何云服务器，所有云端存储均对接用户自有的第三方云盘；开发者不存储任何用户数据，不承担数据泄露或丢失的责任。
4. 多冗余备份：支持同时开启任意多个云盘并行同步，避免单一云盘故障导致数据丢失。
5. 绝不自动删除用户数据：所有删除操作均为软删除，删除内容保留30天；绝对不会自动删除任何用户文件，即使是未被引用的图片。
6. 用户完全掌控数据：数据格式公开透明，支持一键全量导出；应用卸载不会删除任何用户数据，用户可随时迁移至其他软件。
7. 真正的无锁定：提供独立的单网页解密工具，即使软件不再维护，用户也能自己解密所有数据。
三、核心架构要求
1. 数据存储结构（自动创建，用户无需干预）
系统公共文档目录/Documents/[应用名]/
├── Books/  # 所有书的根目录
│   ├── .templates/  # 全局模板目录
│   └── {书UUID}/  # 单本书独立目录
│       ├── Notes/  # 本书加密笔记文件（.note后缀）
│       ├── Assets/
│       │   ├── Images/  # 本书专属图片文件
│       │   └── .trash/  # 图片回收站
│       ├── .templates/  # 本书专属模板目录
│       └── .trash/  # 笔记回收站
└── config.json  # 应用配置文件• 每本书拥有完全独立的存储空间，互相隔离
• 所有文件均存储在系统公共文档目录，应用卸载不删除
• 回收站采用纯文件系统实现，不依赖应用数据库
2. 同步引擎要求
• 文字与图片独立并行同步，文字优先级永远高于图片
• 增量同步：仅传输内容哈希发生变化的文件
• 原子同步：单个文件的同步要么成功要么失败，不会出现半同步状态
• 幂等性设计：重复执行同步操作不会导致数据错误
• 同步失败自动重试，失败原因明确记录并提示用户
• 冲突处理：本地永远是唯一真相源，采用"最后写入胜出"原则
3. 支持的云盘类型
• 第一优先级：WebDAV协议（通用标准，作为基础和兜底）
• 第二优先级：阿里云盘（国内速度最快，作为首推）
• 后续可扩展：OneDrive、天翼云盘等
4. 加密方案（极简透明，零用户感知）
• 采用固定全局256位AES-CTR密钥，硬编码在代码中
• 所有用户、所有设备使用完全相同的加密密钥
• 加密解密过程完全自动，用户无需设置或记住任何密码
• 唯一目标：确保本地和云端文件均为非明文格式
• 架构预留扩展能力，未来可增加可选的用户主密码端到端加密
四、功能需求清单
1. 三级内容管理（严格层级：书→卷→笔记）
• 书（Book）：顶层容器，支持新建、重命名、删除
• 卷（Volume）：书的子容器，支持新建、重命名、删除
• 笔记（Note）：卷的子容器，支持新建、重命名、删除
• 删除规则：所有删除操作弹出确认提示框（仅需点击"确认"按钮，无需输入文字）
• 所有内容按修改时间倒序排列，支持按创建时间排序
• 软删除机制：删除的内容移动到对应书的.trash目录，保留30天，支持恢复和永久删除
• 永久删除需二次确认："永久删除后无法恢复，云端备份也会在下次同步时被移除"
2. 编辑功能
• 支持标准Markdown语法编辑与实时预览
• 无感自动保存：停止输入500ms自动写入本地文件
• 写入原子性：先写临时文件，再重命名覆盖原文件，崩溃不损坏数据
• 支持图片插入：粘贴截图、从相册选择、从文件管理器拖拽插入
3. 图片功能
• 图片压缩：默认将图片转换为WebP格式（质量系数95），仅去除EXIF信息和元数据冗余，不牺牲人眼可感知的画质；设置页提供"保存原图"可选开关（默认关闭，明确标注"开启后大幅增加存储空间和同步时间"）
• 压缩规则：两种模式二选一，绝不同时存储两份图片
• 图片同步：采用15秒惰性批量同步机制
    ◦ 插入图片后延迟15秒开始同步，15秒内替换/删除图片则取消同步任务
    ◦ 15秒内插入多张图片合并为一个批量同步任务
    ◦ 退出当前笔记时强制同步所有未同步图片
    ◦ 退出提示规则：0-3张未同步显示底部Toast，4张及以上弹出确认对话框
    ◦ 未同步图片>5张或总大小>10MB时，用户停止输入5分钟后自动启动提前同步
• 图片加载：本地优先，按需加载
    ◦ 打开笔记时优先加载本地图片，本地不存在则显示占位图并从云端下载
    ◦ 分书按需加载：新设备仅同步所有书的文字内容；只有当用户打开某本书时，才开始后台下载该书的所有图片，未打开的书永远不下载图片
4. 同步功能
• 支持同时添加并开启任意多个云盘账号，所有云盘并行同步
• 每个云盘独立维护同步状态，互不影响
• 首页顶部固定手动同步按钮，点击立即触发全量同步
• 同步过程显示实时进度（如"同步中 3/10"），同步完成后弹出明确提示
• 所有网络环境（WiFi/移动数据）均支持同步，无仅WiFi同步选项
• 支持从任意一个已配置的云盘恢复数据
• 同步失败绝不修改或删除本地数据
5. 全局搜索功能
• 首页顶部唯一搜索入口，支持实时搜索
• 搜索范围切换：搜索栏右侧提供「全部书/当前书」二选一按钮，默认显示「全部书」，切换状态自动保存
• 搜索结果严格按「书→卷→笔记」层级缩进显示，匹配关键词高亮
• 支持点击结果直接跳转到对应位置
• 基于SQLite FTS5全文索引实现，10000篇笔记搜索延迟<100ms
• 保留最近10条搜索历史
6. 自定义模板功能
• 支持创建全局模板（所有书可用）和书内模板（仅本书可用）
• 支持从现有笔记右上角菜单点击"另存为模板"
• 支持在模板管理页面新建空白模板
• 支持编辑、删除模板，删除模板不影响已使用该模板创建的笔记
• 新建笔记时，点击按钮下拉箭头可选择应用任意可用模板
• 模板和普通笔记使用完全相同的加密和同步规则
7. 同步日志与异常处理
• 保留最近30天的完整同步日志，每条日志包含：时间、云盘名称、操作类型、文件名、结果、详细错误信息
• 异常日志用红色高亮显示，常见错误给出明确解决建议
• 冲突情况用橙色高亮记录，并提示用户"该文件存在多个版本，请检查内容"
• 支持一键导出日志文件用于排查问题
8. 数据导出功能
• 单笔记导出：支持导出为明文Markdown、PDF
• 单卷导出：支持导出为ZIP压缩包（含所有笔记和图片）
• 单书导出：支持导出为ZIP压缩包
• 全量导出：支持一键导出所有数据为ZIP压缩包
9. 独立解密工具
• 提供独立的单网页解密工具，与主软件分离部署
• 无需输入任何密码，用户只需拖入.note文件或整个文件夹即可解密
• 工具代码完全开源，支持离线运行
• 在软件关于页和首次启动引导中明确告知用户解密工具的存在和使用方法
10. 基础设置功能
• 云盘账号管理：添加、删除、开启/关闭云盘同步
• 回收站管理：查看、恢复、永久删除内容
• 图片设置：是否保存原图开关
• 模板管理：查看、编辑、删除所有模板
• 关于页面与隐私政策

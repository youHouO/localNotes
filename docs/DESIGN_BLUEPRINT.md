## 三、完整关键交互流程
### 1. 新建书/卷流程
1. 用户点击左侧边栏"新建书"按钮 / 书页面"新建卷"按钮
2. 系统：弹出"新建内容弹窗"，输入框自动聚焦
3. 用户输入名称，点击"创建"
4. 系统：
   - 生成唯一UUID
   - 创建对应目录结构
   - 更新SQLite索引
5. 系统：自动跳转到新建的书/卷页面
6. 系统：将本次变更加入同步队列

### 2. 重命名书/卷/笔记流程
1. 用户点击书/卷/笔记项右侧的「...」更多操作按钮
2. 用户点击菜单中的「重命名」选项
3. 系统：弹出重命名弹窗
   - 标题："重命名书" / "重命名卷" / "重命名笔记"
   - 输入框预填当前名称，文字自动全选状态
4. 用户修改名称，点击「确认」
5. 系统：
   - 更新本地文件/目录名称
   - 更新SQLite索引中的名称字段
   - 将本次变更加入同步队列
   - 底部Toast提示："已重命名"
6. 特殊处理：如果是正在编辑的笔记，编辑页顶部标题同步更新
7. 同步引擎：同步变更到所有已开启的云盘

### 3. 新建笔记与自动保存流程
1. 用户在书/卷页面点击右下角的"+"按钮
2. 系统：在当前卷下创建新笔记
   - 生成唯一笔记UUID
   - 创建临时文件：`Books/{书UUID}/Notes/{笔记UUID}.temp.note`
   - 如果有默认模板，自动填充模板内容
   - 更新SQLite索引
3. 系统：页面跳转到笔记编辑页
4. 系统：用户开始输入内容
5. 系统：用户停止输入500ms后
   - 加密当前内容（生成随机IV → AES-256-CTR加密 → 拼接IV+密文）
   - 写入临时文件
   - 原子重命名临时文件为正式文件：`{笔记UUID}.note`
   - 更新SQLite索引中的修改时间和明文哈希
   - 底部栏状态变为"已保存 ✓"
6. 系统：将本次变更加入文字同步队列（最高优先级）
7. 同步引擎：立即上传该.note文件到所有已开启的云盘
8. 同步引擎：同步完成后更新云端manifest.json

### 4. 图片插入与惰性同步流程
1. 用户在编辑页粘贴截图/从相册选择图片/拖拽图片
2. 系统：
   - 生成唯一文件名：`{时间戳}_{随机数}.webp`
   - 将图片转换为WebP格式（质量95），去除EXIF信息
   - 保存到本地：`Books/{书UUID}/Assets/Images/{文件名}`
   - 在光标处插入Markdown图片链接：`![](./Assets/Images/{文件名})`
   - 更新SQLite索引
   - 立即显示本地图片到编辑区和预览区
3. 系统：启动15秒倒计时
4. 如果15秒内用户删除/替换了该图片：
   - 取消同步任务
   - 不删除本地图片文件
5. 如果15秒内用户没有修改该图片：
   - 将该图片加入图片同步队列（低优先级）
6. 系统：当满足以下任一条件时开始同步图片
   - 同步引擎空闲时
   - 用户停止输入5分钟后
   - 用户退出当前笔记时
7. 同步引擎：上传图片到所有已开启的云盘
8. 同步完成后更新云端manifest.json和本地同步状态

### 5. 全局搜索与结果跳转流程
1. 用户点击首页顶部搜索框
2. 系统：页面跳转到全局搜索页，搜索框自动聚焦
3. 用户输入关键词
4. 系统：实时执行SQLite FTS5全文搜索
5. 系统：将搜索结果按「书→卷→笔记」层级整理
6. 系统：高亮显示所有匹配的关键词
7. 用户点击任意搜索结果
8. 系统：
   - 如果是书籍结果：跳转到该书的首页
   - 如果是卷结果：跳转到该卷的内容列表页
   - 如果是笔记结果：跳转到该笔记的编辑页
9. 系统：将本次搜索关键词加入搜索历史

### 6. 从回收站恢复笔记流程
1. 用户点击左侧边栏的"回收站"入口
2. 系统：页面跳转到回收站页
3. 系统：扫描所有书的`.trash`目录，显示所有已删除的内容
4. 用户点击某条内容的"恢复"按钮
5. 系统：
   - 将内容从`.trash`目录移回原位置
   - 更新SQLite索引
   - 将本次变更加入同步队列
6. 系统：底部Toast提示"恢复成功"
7. 同步引擎：同步恢复的文件到所有已开启的云盘

### 7. 多设备首次登录从云端恢复数据流程
1. 用户在新设备打开应用
2. 系统：显示首页空数据状态
3. 用户点击"从云盘恢复已有数据"
4. 系统：弹出云盘添加弹窗
5. 用户输入云盘地址、用户名、密码
6. 系统：连接云盘并验证
7. 系统：拉取云端的`manifest.json`文件
8. 系统：
   - 下载所有`.note`文件到本地对应目录
   - 不下载任何图片文件
   - 扫描所有`.note`文件，重建SQLite索引
9. 系统：显示所有书的列表和文字内容
10. 用户打开某一本书
11. 系统：开始在后台下载该书的所有图片文件
12. 图片下载完成后自动显示在笔记中

### 8. 手动同步与后台同步区分流程
1. **场景A：用户手动点击顶部同步按钮**
   - 弹出同步进度弹窗，显示实时进度
   - 用户可点击"后台同步"关闭弹窗，同步继续在后台进行
   - 同步完成后弹窗自动关闭，底部Toast提示"同步完成"
2. **场景B：后台自动同步（编辑后/定时）**
   - 不弹出任何弹窗，仅更新底部状态栏的同步状态图标和文字
   - 同步完成后底部状态栏显示"最后同步: XX:XX"
   - 同步失败时底部状态栏显示红色图标，点击可查看同步日志
